JS Numbers

Unlike many other programming languages, JS does not define diffent types of numbers, like intigers, short, long, floating- point.
JS numbers are always stored as double precision floating point numebrs, the interbational IEEE 745 standard.
This format stored numbers in 64 bits, where the number - the fraction- is stored in bits 0 to 51, the exponent in bits 52 to 62, and the sign bit 63:
Value:52 bits (0- 51)
Exponent:11 bits(52-62)
Sign:1 bit(63)

1. Intiger precision

3. Adding numbers and strings

WARNING !!
JavaScript uses the + operator for both addition and concatenation.
Numbers are added. Strings are concatenated.

When adding two numebrs the result will be a number:
let x = 10;
let y = 20;
let z = x + y ;

The result will be 30;

If you add two strings, the result will be a string concatenation:
let x = "10";
let y = "20";
let z = x + y ;

The result will be 1020;

If you add a number and a string, the result will be a string concatanation:
let x = 10;
let y = "20";
let z = x + y ;

The results will be 1020;

If you add a string and a number, the result will be a concatenation:
let x = "10";
let y = 20;
let z = x + y ;

The results will be 1020;

let x = 10;
let y = 20;
let z = "The results is "x + y ;

The results will be 1020;

4. Numeric Strings

JS will try to convert strings to numbers in all numeric operations:
JS strings can have numeric content:

This is ok:

let x = 100;
let y = "100";

This is ok:

let x = "100";
let y = "10";
let z = x / y;

10

This is ok:

let x = "100";
let y = "10";
let z = x * y;

1000

This is ok:

let x = "100";
let y = "10";
let z = x - y;

90

This is not ok:

let x = "100";
let y = "10";
let z = x + y;

10010

In the last example, the "+ " is used by JS to concatenate strings.

NaN - Not a number

NaN is a JS reserved word indicating that a number is not a legal number.
Trying to do arithmetic with a non- numeric string will result in NaN.
You can use the global JavaScript function isNaN() find out if a value is a not a number.

let x = 100 / "Apple";
as 100 is divided to a string - the results will be NaN.
