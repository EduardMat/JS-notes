JS Objects

Real life objects, properties and methods

In real life, a duck is an object.
A duck has properties like weight and color, and methods like eat and fly:

Objects


Properties


pet.name = Duck;

pet.model = fluffly;

pet.weight = 3 kg;

pet.color = green;

Methods


pet.fly();

pet.run();

pet.eat();

pet.sleep();

All ducks have the same properties, but the property values differ from duck to duck.
All ducks have the same methods, but the methods are performed at different times.

JS objects

We have already learned that JS variables are containers for data values.
This code assigns a simple value to a variable called car:

Example:

let car = "Fiat";

Objects are variables too.But objects can contain many values.
The code assigns many values (green, fluflly ) to a variable named duck:

Example:

const car = {model: "fluffly", weight:"3 kg", color:"green"};

The values are writted as name:value pairs (name and value separated by a colon)

It is a common practice to declare objects with the const keyword.
Learn more about using const with objects in the chapter: JS Const.

Object Definition

You define (and create) a JS object with and object literal:

Example:

const person = {firstName:"John", lastName:"Doe", age:50, eyeColor:"blue"};
// Display some data from the object:
document.getElementById("demo6").innerHTML =
person.firstName + " is " + person.age + " years old.";
