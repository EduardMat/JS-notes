JS String

    JS strings are for storing and manipulating text.
    A JS string has zero or more characters written inside quotes.

Example:

 let  text = "John Doe";
 script>
let text = "John Doe";  // String written inside quotes
document.getElementById("demo").innerHTML = text;
/script>
  

You can use single or double quotes.
  You can use quotes inside  a string , as long as they don t match the quotes surrounding the string:
  let carName1 = "Volvo XC60";  // Double quotes
  let carName2 = 'Volvo XC60';  // Single quotes

let answer1 = "It's alright";
let answer2 = "He is called 'Johnny'";
let answer3 = 'He is called "Johnny"';

Escape Character

  Because strings must be written within quotes, JS will misunderstand the string:
  let  text = "We are so- called "Vikings" from the north";

  The string will be chopped to "We are the so-called". 
  The solution to avois this problem, is to use the backslash escape character.
  The backslash (\) escape character turns special characters into string characters:
CODE-- -RESULTS-- -DESCRIPTION--
\ ` ` single quote
\ " " double quotes
\\ \ backlash 
  1.The sequence  \"  inserts a double quote in a string:
       let  text =  "We are the so-called \"Vikings\" from the north.";==> We are the so-called "Vikings" from the north.

  2.The sequence  \ ` inserts a single quotes in a string:
       let  text =  " It is \ ` alright";==> It's alright.

  3.The sequence \\ inserts a backslash in the string:    
       let  text =  "The character \\ is called backslash";==> The character \ is called backslash.

Six other escapes are valid in JS:

Code----|Results
\b==>backspace
\f==>form feed
\n==>new line
\r==>carriage return
\t ==>horizontal tabulator
\v ==>vertical tabulator

The 6 escape characters above were originally designed to control typewritters, teletypes, and fax machines.They do not make any sense in HTML.

Breaking long code lines

    For best readibility, programmers often like to avoid lines longer than 80 characters.

    1.If a JS statement does not fit on one line, the best place to break it is after an operator:

Example:

    document.getElementById("demo").innerHTML =
   "Hello Dolly!";

2.You can also break up a code line within a string with a single backslash:

Example:

    document.getElementById("demo").innerHTML =  "Hello \

   Dolly!";

The \ method is not the prefferd method.It might not have the universal support.Some browsers do not allow spaces behind the \ .

3.Another safe way to break up a string, is to use string addition:

Example:

    document.getElementById("demo").innerHTML = "Hello " +
 "Dolly!";

!!!You cannot break up the code with a backslash: This is wrong!!!

Example:

    document.getElementById("demo").innerHTML = \
 "Hello Dolly!";

JS String as objects 

  Normally, JS strings are primitive values, created from literals:
   let  x = "John";
But strings can also be defined as objects with the keyword  new:
let  y =  new String John;

Example:

let x = "John";                ==>this is a string 
let y =  new  String("John");    ==> this is an object  

  Do not create string objects.
  The  new  keyword complicates the code and slow down the execution speed.
  String objects can produce unexpected results.

  When using the ==  operator, x and y  are equal:
   let x =  "John";
   let y = new  String ("John")
